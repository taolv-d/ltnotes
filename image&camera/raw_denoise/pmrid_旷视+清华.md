### 1. 核心思想：手机传感器的噪声是可测量的

论文最关键的观点是：**与其训练一个通用的大模型，不如针对具体手机的传感器，量身定制一个小模型**。

论文基于**泊松-高斯噪声模型**来描述RAW图像的噪声。简单说，噪声主要来自两方面：

- **散粒噪声**：与进光量有关，光越强，噪声波动越大。
    
- **读出差分噪声**：与传感器电路有关，和进光量无关，但和ISO（增益）强相关。
    

**测量方法很巧妙**：  
对着灰度卡连拍64张，算出每个像素位置的**均值**（当作真值）和**方差**（当作噪声强度）。通过线性回归，就能拟合出不同ISO下的噪声参数 `k` 和 `σ^2`。

这是论文最精妙的理论部分，目的是解决**不同ISO下噪声强度差异巨大**的问题。

- **难点**：ISO 100 和 ISO 6400 的噪声完全不是一个量级。如果直接喂给一个网络，网络会很困惑，难以收敛。
    
- **做法**：设计了一个数学变换 `f(x)=x/k​+σ^2/k^2`。
    
- **效果**：经过这个变换后，**无论原始ISO是多少，噪声的分布都会被映射到一个统一、稳定的空间**(即方差`==`均值)。在这个空间里，ISO 100和ISO 6400的图像看起来“噪声水平差不多”。
    对比VST 方差~=1

这样一来，**一个很小的网络就能处理所有ISO的降噪任务**，而不用为每个ISO单独训练模型。

k-Sigma 是线性变换，VST 是非线性变换[[VST]]

k-Sigma 的局限：
1.  需要标定
2. 公式9 将泊松分布近似为高斯，但这再极暗光条件不成立，泊松分布不是左右对称的，VST 变换能够更好处理这种情况
3. k-Sigma 仍然没有把不同增益拉到完全相同的水平下，即不同增益变换后的方差仍然不是完全相同的，这对网络的要求更高。相反VST让方差变为1，网络更容易学去噪，YOND中用VST变换+简单网络实现了更好的泛化性[[YOND]]

### 4. 网络架构：极度轻量化

PMRID 的网络结构就是“RAW 4通道输入 + ISO噪声归一化 + 轻量残差U-Net + 深度可分离卷积 + 全局残差输出”。

- **基础框架**：4层编码器+4层解码器的 **U-Net** 结构。
    
- **计算瘦身**：大量使用**深度可分离卷积**[[../../machine learning/nn积木/DSC|DSC]]，只在输入输出层用普通卷积。
    

先说输入。原始 Bayer 图先被打包成 RGGB 四通道，原来单通道 H x W 的 Bayer，会变成 4 x (H/2) x (W/2)。这样做的好处是既保留 RAW 的马赛克结构信息，又把空间尺寸减半，计算量更适合移动端。推理前还会做一次基于 ISO 的噪声归一化 KSigma，把不同 ISO 的噪声分布拉到一个统一参考域里，再乘上 256 送入网络。

网络主干是标准“编码器-解码器+跳连”结构：

1. conv0：4 -> 16，普通 3x3 卷积，提浅层特征
2. enc1：16 -> 64，下采样 2 倍，含 2 个残差块
3. enc2：64 -> 128，再下采样 2 倍，2 个块
4. enc3：128 -> 256，再下采样，4 个块
5. enc4：256 -> 512，最深层，再下采样，4 个块
6. encdec：把瓶颈层 512 通道压到 64
7. dec1~dec4：逐级反卷积上采样，并和编码器对应层做相加式 skip fusion
8. out0 + out1：再做一次残差块和最终 3x3 卷积，输出 4 通道噪声修正量
9. 最后 pred = inp + x，是全局残差学习

如果把尺度写出来，会更直观。设 packed RAW 输入是 4 x H x W，那么特征图尺寸大概是：

- conv0: 16 x H x W
- enc1: 64 x H/2 x W/2
- enc2: 128 x H/4 x W/4
- enc3: 256 x H/8 x W/8
- enc4: 512 x H/16 x W/16
- dec1: 64 x H/8 x W/8
- dec2: 32 x H/4 x W/4
- dec3: 32 x H/2 x W/2
- dec4: 16 x H x W
- 输出: 4 x H x W

它的每个编码块和解码块都很“移动端友好”。关键点是大量使用了 depthwise separable convolution[[../../machine learning/nn积木/DSC|DSC]]，也就是深度可分离卷积。EncoderBlock 里是两层 5x5 可分离卷积加残差支路；DecoderBlock 里是两层 3x3 可分离卷积加残差。整个网络没有 BatchNorm、没有 attention、没有 transformer，设计目标非常明确：在 RAW 域做有效降噪，同时尽量省算力、省参数。

还有两个很值得注意的实现细节：

- 跳连融合用的是“相加”不是拼接。DecoderStage 先把上采样结果和 skip 特征各自映射到同一通道数，再直接 x + y。这比 U-Net 常见的 concat 更省显存和算力。
- 输入会 pad 到 32 的倍数。原因是 packed 后网络内部总共下采样了 4 次，也就是 2^4 = 16 倍；而 packed 本身又相当于对原始 Bayer 空间缩了一半，所以对应原始 Bayer 尺度就是 32 的对齐需求。
