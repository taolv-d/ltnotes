# 0 dtof系统
## dtof系统
**VCSEL:** VCSEL全称为垂直腔面发射激光器（Vertical-Cavity Surface-Emitting Laser），是半导体激光器的一种，当前以砷化镓半导体为基础材料的VCSEL居多，发射波长主要为近红外波段。

**TDC:** 记录start信号(一个)和stop信号(多个)之间的时间，并将时间量化为数字量（编码为code）

**DCR：** 没有光时SPAD产生的信号（环境光引起），因此需要多次打光来处理背景噪声（参考下面的直方图算法）

# 1 dtof 系统
**dtof与itof区别：** 
	itof （indirect）间接飞行时间法，测量距离与调制频率成反比，但是测距精度高（mm级）
	dtof （direct） 直接飞行时间法，使用雪崩二极管，测量距离远（几米--200米）
**FSI与BSI工艺:**
	FSI: SPAD，TDC，memory等都在一层，芯片面积比较大
	BSI: 堆叠，分为模拟带（SPAD readout TDC）,逻辑带（存储、控制（spad开关、帧率、积分时间、功率等）、算法、IO）
**工作模式：**
Flash: 所有pixel可以同时打开（需要每个pixel一个存储、处理），帧率高，但分到分到每个pixel的光功率低，造成测试距离近，几十米
扫描：一次只能打开一行（只有一行TDC、处理电路等）。
*binning:* 一个大pixel里面合并多个小pixel, 小pixel公用一个TDC,提高SPAD被触发的概率
![[attachments/Pasted image 20260512223858.png]]
**dtof系统的挑战：**
	1、SPAD 尺寸，需要高压，因此尺寸很大，分辨率低
	2、片上直方图算法：工艺成本、直方图存储成本（时间换空间来压缩）
	3、帧率
	4、精度
	5、抗环境光干扰能力
**抗环境光技术**
*空间相关技术：* 两个（多个）空间上很近的SPID通过与门连接，只有都有有效输出时才是真实信号，否则是干扰
*滤波技术：* 设计 核函数

# 2 发射端建模
发射端通过驱动器驱动激光器（VCSEL）发射脉冲光信号（窄脉冲）
难点：1、窄脉冲，2大功率
**窄脉冲发生：**
生成两个脉冲信号，信号之间有延时，两信号相异或得到窄脉冲输出。延时有delay line 器件（多级或门、反相器等组成），控制delay line级数来控制脉冲宽度
![[attachments/Pasted image 20260512223904.png]]
**大功率——多通道发射：**
激光器、接收端分成多个bar, 两边依次打开（一行激光器对应一行接受，提高能量利用率），如下图中相同颜色的发射接收同时工作，上下扫描。
![[attachments/Pasted image 20260512223910.png]]
![[attachments/Pasted image 20260512223931.png]]
**建模**
基于高斯分布的激光脉冲光子数分布构建发射端模型
1、计算脉冲里的能量（输入平均光功率、占空比、频率 计算peak power）
2、根据能量、光波长（通常940nm）得到光子数
3、光子数根据高斯分布得到每个小时间单元内的光子数量

# 3 接收端
**SPAD:** 单光子雪崩二极管，工作在盖革模式时（加大的反向偏压23V左右，增益认为是无穷大），接收到单光子时有概率引发雪崩。
**甄别器：** SPAD 存在噪声，需要一个阈值将高于阈值的信号筛选出来并整型

**SPDA淬灭技术：** SPAD 不能一直雪崩，否则无法接收到下一个光信号
	1、被动淬灭：二极管上串大电阻，电流大时电阻分压，SPAD电压不够，会发生淬灭。缺点淬灭时间长，由RC常数决定。大面阵常用
	2、主动淬灭：反馈电路接收到脉冲，反馈调节二极管电压
	3、门控淬灭：通过脉冲信号控制SPAD两端电压，只有门控信号有效时，SPAD才能工作
![[attachments/Pasted image 20260512223953.png]]
**SPAD性能参数**
1、雪崩电压：不同温度下的雪崩电压
2、探测效率：出发雪崩的概率，与雪崩电压有关（电压越高，越灵敏，不同波长是否也有差异）
3、暗技术：不加光时触发雪崩。实际就是噪声（DCR）
4、死时间：雪崩发生到淬灭的时间（可能损坏SPAD，无法探测）
5、后脉冲
6、填充因子

**PDE** 是不同波长下的入射光子触发雪崩的概率

SPAD建模：
10M 激光器发射频率（1s 打10M个脉冲光）
DCR=100pcs（时间轴上均匀分布100个DCR 噪声）
PDE=1% , 对应1s 有100K个信号，对应每次激光器重复频率后的位置是固定+小抖动
每个小周期内可能有多个脉冲，每个脉冲发生的时间量化为数字，一个脉冲器件内就是一个只有几个bin的直方图。所有脉冲直方图叠加就是这段时间内的直方图
# 4 TDC
计算开始时间到结束时间的电路，并量化为数字编码（类似定时器？）
**粗计数：** 根据时钟脉冲计数（需要皮秒级计数，对应GHz,功耗较高，且电路复杂，因此不能通过提高时钟的频率来计数）
**细计数：** 用于处理小于一个时钟脉冲的时间，使用多个不同级数的延时链来实现
**量化时间：** 粗计数+细计数

# 9 SPAD 如何成灰度图像（统计光子数）
问题：
**死区**时间内无法检测光子，造成光很亮的时候（一直不能淬灭再恢复，新来的光子无法再产生的脉冲）
![[attachments/Pasted image 20260512224035.png]]
**计数器有上限**造成光子数过多的时候饱和
	一种巧妙的方法：正常计数器例如8bit，最大只能记数到256，对应一帧只能记录256个光子。一种巧妙的方法，将一帧的时间分为不同长度的时间片段，找到一个小时间片段内的计数再256内。那么整个一帧的计数就是 小时间片内计数亮 × 时间片数量
