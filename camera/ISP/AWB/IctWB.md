1. 白点法（White Patch / Max-RGB）

原理：假设图像中最亮的像素就是白色，gain = 1 / max(R,G,B)。

不用的原因很简单——太脆弱了，完全不适合严肃的 RAW 处理：

- 🌻 一张黄花照片 → 最亮像素是黄色 → 白平衡完全错误
- 🌅 日落 → 最亮像素是橙红色 → 严重偏蓝
- 🚗 任何带高光的彩色物体 → 直接失效

没有任何专业 RAW 软件把白点法作为 AWB 方案。它只在特定受控场景下有用（如实验室、均匀照明），对自然摄影基本是废的。

---
2. 灰边法（Gray Edge）

原理（van de Weijer et al., 2007）：假设场景中边缘梯度差的平均值为灰色：

$$\int \left| \frac{\partial f(x)}{\partial x} \right|^p dx = k \cdot e$$

相比灰世界，它解决了大面单色场景的问题，但也有自己的缺陷。

RT 不用的原因是：它带来的价值已被 ItcWB 覆盖了。

来看三种算法的失效场景对比：

灰边法解决的核心问题是"大面单色场景下灰世界
根本不依赖场景平均颜色，它直接匹配图像中的

- 灰世界在大片绿草地上偏品 → 灰边法能解决 → 但 ItcWB 也能解决
- 灰边法在边缘有强偏色时也失效 → ItcWB 不受

灰边法是"改进版的灰世界"，而 ItcWB 走的是完全不同的技术路线（颜色匹配 vs 灰度假设），天然覆盖了灰边法的优势场景。

RawTherapee AWB 算法：ItcWB（温度相关性）

作者 Jacques Desmis (2018-2023)，Ingo Weyrich (2020)。位于 rtengine/rawimagesource.cc:5291，约 2000 行。

🎯 核心思想

不是传统的"灰色世界"假设（对 RGB 通道直接求平均）。而是 温度相关性方法：将图像中的颜色分布与 265 种已知光谱参考色在不同色温下的理论色度进行比较，找出相关性最高的温度，从而反推 RGB 增益。

---
📊 算法流程（六步）

第一步：准备参考光谱色

265 种光谱色 → 在 191 个色温(2000K~15000K)下 → 计算 xy 色度坐标
- 光谱色来源：ColorChecker24、JDC 468 色靶、花卉网络数据等，5nm 步长
- 每个色温都有对应的 Xwb、Zwb 白点值（Txyz[191] 表）
- 色温步长不等（低温密、高温疏）

第二步：图像采样与降噪

原始 RAW → 1/3 亚采样 → 3x3 中值滤波降噪
→ 用相机色温的 RGB 乘法器 → 转换到 xyY 空间
- precision = 3（每 3 个像素取 1 个）
- 中值滤波去噪后 clip
- 用当前相机色温对应的 rmm[repref]/gmm[repref]/bmm[repref] 转换 RGB

第三步：构建 xy 直方图并匹配光谱色

图像 xy 值 → 2D 直方图（236 个 bin）
→ 按出现次数排序 → 取 Top 50~70 种颜色
→ 计算每种颜色的色度 = sqrt((x - x_white)^2 + (y - y_white)^2)
→ 在光谱参考色中找 deltaE 最近的匹配
关键点：选择的是出现频率高的颜色（histnum

第四步：温度扫描与 Student 相关性

for (int tt = ttbeg; tt < ttend; ++tt) {  /
    // 对每个色温 tt：
    //   1. 用 rmm[tt], gmm[tt], bmm[tt] 将图像颜色转回 xyY
    //   2. 重新计算光谱参考色在该色温下的

    float abstud = fabs(studentXY(image_xy,tt));

    if (abstud < minstud) {

在最佳色温附近 → 扫描 134 个 green 值 (0.4
→ 每个 green + 附近色温 → 重新计算乘法器 →

temp → multipliers 的核心流程（temp2mul() a

1. 二分搜索色温，每次用 temp2mul() 计算对应
2. 比较 tmpb/tmpr 与目标 bmul/rmul，收敛温度
3. 反算 green = (tmpg/tmpr) / (gmul/rmul)

简单距离的问题：

方法 A：直接算 |x̄_image - x̄_ref|

  场景1（颜色丰富）：|0.35 - 0.33| = 0.02
  场景2（单色场景）：|0.35 - 0.33| = 0.02

  但场景1有 50 种颜色，0.02 的偏移很显著
  场景2只有 5 种颜色，0.02 可能只是采样噪声

  简单距离无法区分！

t 统计量的优势：

方法 B：t 统计量 = 均值差 / 标准误

  场景1：均值差 = 0.02, 方差小(颜色多且集中
         → |t| 大 → 判断为不匹配

  场景2：均值差 = 0.02, 方差大(颜色少且分散
         → |t| 小 → 判断为"证据不足，可能是
