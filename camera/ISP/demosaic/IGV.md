
IGV 全称是 Integrated Gaussian Vector on Color Differences。它非常像 [[RCD]]：

```text
1. 先在 R/B 点恢复 green
2. 把色差 G-R / G-B 作为主要传播对象
3. 根据局部色差稳定性给方向权重
4. 沿更平滑方向传播 chroma
```

## 1. 插值 G

**第一步先直接分离 CFA 到RGB三个通道，此时三个通道还是稀疏的**
工程实现是这样的，但是对算法流程其实不是必要的

**第二步在 R/B 点生成 vdif / hdif（水平、垂直色差）**

先构造上下左右四个方向的梯度，例如N方向：
```cpp
ng = eps + abs(G[N1] - G[N3]) + abs(C[x]  - C[N2])
// 意思是 G 通道的梯度，或者 C 通道的梯度过大，说明梯度不平滑，不适合插值G
```

计算上下左右方向的 高阶 green 候选估计，例如N：
这里用多个位置G采样支持，同时用更远的 R/B 矫正。这里用了更多的像素，目的就是更少噪声
```cpp
nv =
(
  23 * G[N1]
+ 23 * G[N3]
+      G[N5]
+      G[S1]
+ 40 * C[x]
- 32 * C[N2]
-  8 * C[N4]
) / (48 * 65535)
```


接着做正交方向融合，再转成色差：
这里利用对侧梯度加权，目的就让梯度变化大的一侧权重更低
```cpp
vdif = (sg * nv + ng * sv) / (ng + sg) - C[x]
hdif = (wg * ev + eg * wv) / (eg + wg) - C[x]
```

所以到这一步，IGV 得到了水平、竖直两个方向的色差

**第三步融合水平竖直两个色差**
这里简单理解就是看 水平、竖直 两个方向差值出来的色差波动情况（利用方差等），不稳定的方向给更低的权重

在真正融合前，IGV 还会先对 `vdif` / `hdif` 做一次受 median 约束的局部平滑，目的是限制异常值，并平滑色差：

```cpp
nv = median(
    0.725  * vdif[x]
  + 0.1375 * vdif[up2]
  + 0.1375 * vdif[down2],
    vdif[up2],
    vdif[down2]
)
```

最后的融合也是经典的方案：

```cpp
chr[d][indx] = (eg * nv + ng * ev) / (ng + eg);
```

有了色差，就能恢复最新的G：

```text
G = C + (G-C)
```

## 2. 在 R/B 点补另一个颜色

此处的差值直接用四个对角方向传播 chroma。R 的四个斜对角都是B，反过来B也一样。
方法也是权重、初步估计、加权最终估计：

先构造四个对角方向权重，也是色差变化越大，权重越低：

```text
weight = 1 / (eps + 该对角方向上的色差变化)
```

然后对每个方向的候选色差做 median 限制，相当于中值滤波

最后也是四个方向加权求和，也是色差变化大的权重低：

```cpp
chr[c][indx] =
    (nwg * nwv + neg * nev + swg * swv + seg * sev)
  / (nwg + neg + swg + seg);
```

## 3. 在 G 点补 R/B 色差

到这一步，每个G的上下左右四个方向的R B 都已经有了。算法也是用上下左右四个加权，过程跟上一步类似，只是上一步用的斜向四个。

这一步计算完成后 R G B三个通道就完成了
