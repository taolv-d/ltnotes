| 算法     | 核心策略              | 优点      | 代价     |
| ------ | ----------------- | ------- | ------ |
| Linear | 固定邻域加权平均          | 快、简单    | 边缘和彩边差 |
| VNG    | 8 方向梯度筛选          | 适配性强    | 质量有限   |
| PPG    | 梯度插绿，色差补红蓝        | 质量和速度折中 | 仍偏传统   |
| AHD    | 水平/垂直候选 + Lab 同质性 | 边缘较好    | 计算量大   |
| DCB    | 方向图、多轮绿修正、色差插值    | 细节较强    | 管线复杂   |
| DHT    | 亮度比例方向图           | 对曝光比例敏感 | 复杂     |

## Linear

Linear 的核心思路是：对每种 CFA 相位预先生成一组邻居访问规则，然后对缺失颜色做加权平均。

从算法原理看，Linear 假设局部图像是低频且平滑的：一个缺失颜色值可以由周围最近的同色采样平均得到。它不试图区分边缘方向，也不利用红、绿、蓝之间的相关性，只把 CFA 当作三张被稀疏采样的单通道图像分别补齐。

它在平坦区域通常没问题，但在斜边、细线和高频纹理上容易把边缘两侧的颜色混在一起，形成彩边或 false color。

## VNG：按梯度选择方向的平均

VNG 的全称是 Variable Number of Gradients：

1. 先用 Linear 得到一个初始完整 RGB 图。
2. 对每个像素计算 8 个方向的梯度。
3. 选择梯度较小的方向，沿这些方向平均。
4. 用颜色差修正当前像素缺失颜色。

VNG 比 Linear 强，不如后续的色差分离方案

## PPG：色差分离

PPG 是 Patterned Pixel Grouping：

1. 先填非绿像素位置的绿通道。
2. 在绿像素位置填红/蓝。
3. 在红像素填蓝、蓝像素填红。

它把去马赛克拆成两个层次：先尽量准确恢复绿色亮度层，再用绿色层去引导红蓝色度层。这个拆法来自一个经验事实：自然图像中亮度高频细节主要落在绿色通道，而红绿差、蓝绿差这类色差信号通常更平滑（类似YUV中UV分量可以压缩很小）。

```
R = G + interpolate(R - G)
B = G + interpolate(B - G)
```

这样做的好处是边缘主要由 G 控制，红蓝不会完全独立扩散。缺点是如果绿色已经插错，红蓝会跟着被带偏；如果场景里有强饱和彩色细纹，色差平滑假设也会失效。

## AHD：生成水平/垂直两个候选图，再按同质性选择

AHD 是 Adaptive Homogeneity-Directed interpolation：

1. 构造两个完整 RGB 候选：
   - 水平方向插值候选。
   - 垂直方向插值候选。
1. 把两个候选转换到近似感知均匀的 CIELab 空间（也可以是YUV）。
2. 在局部邻域内计算哪个候选更“同质”。
3. 每个像素选择同质性更高的候选；如果接近，就平均。

AHD 假设：正确的插值方向会让局部区域在感知颜色空间中更连续、更同质；错误方向通常会跨过边缘，把两侧颜色混合，导致 Lab 空间里的局部差异增大。

AHD 通常比 PPG 更能抑制彩边，因为它显式惩罚色度不连续；但它也可能在细小重复纹理上选错方向，产生 zipper 或局部伪色。

计算量更大：两份候选图、Lab 转换和同质性统计


## DCB：方向图 + 多轮绿通道校正 + 色度插值

DCB 也以绿色通道为核心，它反复迭代把绿色通道搞精细，然后再搞 红-绿 蓝-绿色差图

DCB 可以拆成三个互相依赖的模型：

1. 绿色层是亮度和细节主体。先把 Bayer 中缺失的绿色补出来，并尽量沿真实边缘方向插值。
2. 方向不是孤立像素属性，而是局部方向场。方向图由当前绿色层计算出来，然后又反过来修正绿色层。（绿色层越准，方向图越准；方向图越准，下一次绿色修正越准）
3. 红蓝色度相对低频。通过 `R-G`、`B-G` 这类色差信号恢复，再加回绿色层。

完整流程比较长，建议看 libraw 源码

```
第一阶段：分别生成水平和垂直绿通道候选：
	水平候选认为左右两个绿色更可靠，垂直候选认为上下两个绿色更可靠。它还没有真正解决方向问题，只是准备两个可比较的候选。

第二阶段：给两个候选补红蓝色差。
	目的是: 让两个候选图都具备完整 RGB 信息，方便后面判断哪个绿色方向更合理

第三阶段：比较原始局部结构与两个候选图的局部结构，决定当前像素绿通道采用水平候选还是垂直候选：

第四阶段：迭代修正绿色层
	1. 先优化绿色：如果红/蓝真实采样显示这里有高频变化，绿色也应该保留对应的高频结构，而不是被简单平均糊掉。
	2. 更新方向图：方向图的判断只看绿色层：如果当前绿色高于上下左右平均，就比较水平邻居和垂直邻居哪一侧更像同一结构；如果当前绿色低于平均，也做对应的反向比较。利用局部极值关系判断该像素更应该沿水平还是垂直传播
	3. 用新的方向图重新插绿色。

第五、六阶段：基础红蓝重建和绿色后修正 + 方向修正。

第七阶段：恢复红蓝备份并重新计算最终红蓝。

第八阶段：可选增强。
```


## DHT：亮度比例判断方向

 
DHT **用“比值距离”而不是“绝对差”衡量亮度变化**（假设局部颜色比值比绝对差更稳定）。摄影里更自然的亮度单位是曝光 stop：`10 -> 20` 和 `1000 -> 2000` 都是加一档，而普通绝对差会认为后者大得多。

```text
缺失绿色 ≈ 当前红/蓝 × 邻近绿色与邻近红/蓝的比例
缺失红/蓝 ≈ 当前绿色 × 邻近红/蓝与邻近绿色的比例
```

明暗变化剧烈但颜色相对稳定的区域可能更合理；
但在低亮度、噪声或接近黑电平的位置，比值模型会更敏感，所以代码里需要 float 缓冲、最小值保护、hot pixel 隐藏和上下界压制。
