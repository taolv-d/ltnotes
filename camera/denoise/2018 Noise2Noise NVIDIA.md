原文：Noise2Noise: Learning Image Restoration without Clean Data

本文提出**无需干净数据即可实现高质量的图像还原**。**仅用含噪图像对进行训练**，达到甚至超越传统依赖干净数据的去噪性能。

# 结论及推导

**本文结论：不需要干净图，只需要同一个场景的两张独立噪声图，就可以训练出去噪网络。**
这里的关键是“**独立噪声**”——两张图分别是在同一场景下两次独立拍摄的，因此它们包含的信号相同（场景不变），但**噪声不同**（噪声是随机的，每次采样的值不一样）。

**噪声建模**
- $S$ 是真实场景，我们永远不知道它。
- $x_1=S+n_1$，第一张噪声图，噪声$n_1$是随机采样。
- $x_2=S+n_2$，第二张噪声图，噪声$n_2$是随机采样，且**与 $n_1$ 统计独立**。

**网络目标**：我们训练一个网络$f$，**输入$x_1$，让它去预测$x_2$**

**L2 损失**：
loss 为L2，也就是最小化$||f(x_1)-x_2||^2$ 。这个损失函数对所有可能的$(x_1, x_2)$的期望值为：
$$
\begin{split}
E[||f(x_1)-x_2||^2]
&=E[||f(x_1)-S-n_2||^2]\\
&=E[||f(x_1) - S||^2] - 2E[(f(x_1) - S)n_2] + E[||n_2||^2]
\end{split}
$$
由于$n_2$与$x_1$（以及 $f(x_1)$）在统计上是独立的，且$n_2$的期望值为 0（噪声0均值假设），则交叉项为0:
$$
E[(f(x_1) - S)n_2]=0
$$
因此损失函数等价于：
$$E[||f(x_1) - S||^2] + E[||n_2||^2]$$
其中第二项为常数。因此：**无论 f 是什么，最小化这个损失等价于最小化 f(x₁) 与不可见的真实信号 S 之间的 L2 距离。** 即学习降噪。



**与最小二乘的关系（个人理解）**

Noise2Noise 基于 L2 loss（均方误差， 关于L2更深入的介绍见：[[../../ML&CV/loss指标/M-估计量|M-估计量]]），跟最小二乘法是有关系的（最小二乘法也是 最小化“预测值”与“真实值”之间差值的**平方和**）。
- 最小二乘法 当有无穷多带噪数据时，可以得到真实分布的无偏估计。但实际使用智能得到真实分布的最优估计（无法提供无穷多数据）。
- Noise2Noise 这里也是一样，**无法完美估计出真实数据**。

# 哪些噪声可以用

| 噪声类型                  | 适用损失       | 常见产生条件 / 典型场景                  | 是否适用 Noise2Noise |
| --------------------- | ---------- | ------------------------------ | ---------------- |
| 高斯噪声                  | L2​        | 传感器热噪声、低光环境下CMOS读噪声、通信信道噪声     | ✅                |
| 泊松噪声                  | L2​        | 光子计数不足（如低光摄影、天文成像、医学低剂量CT）     | ✅                |
| 随机像素丢失（伯努利）           | 加权 L2​     | 数据包丢失、老旧存储介质损坏、随机采样传感器（如单像素相机） | ✅                |
| 离群点（如随机文本覆盖）          | L1​        | 图像被随机位置、颜色、字号的文本或Logo覆盖        | ✅                |
| 脉冲噪声（随机颜色替换）          | 退火 L0​     | 传感器像素故障、传输中的比特翻转、老旧数字图像存储错误    | ✅                |
| 蒙特卡洛渲染噪声              | 自定义 LHDR​  | 路径追踪中采样不足（低 spp，如实时渲染、动画帧渲染）   | ✅                |
| MRI 欠采样（k-space 随机保留） | L2​ + 频域约束 | 磁共振成像加速扫描（压缩感知MRI），减少患者扫描时间    | ✅                |
| 饱和/剪切噪声               | ❌ 不适用      | 相机传感器过曝（高光剪切）、ADC量程上限截断        | ❌                |

# 哪些loss可用

涉及到需要跟真值比较误差的

| 损失类型     | 示例                   | 是否需要干净图像        | 能否用于 Noise2Noise |
| -------- | -------------------- | --------------- | ---------------- |
| 像素级（均值型） | L2, L1, Huber        | ❌ 不需要（只需噪声期望正确） | ✅                |
| 像素级（众数型） | 退火 L0                | ❌ 不需要           | ✅（近似）            |
| 感知/结构损失  | SSIM, LPIPS, MS-SSIM | ✅ 需要            | ❌                |
| 对抗损失     | GAN, WGAN            | ✅ 需要（判别器需要干净分布） | ❌                |
# 训练与应用

**训练**
- **网络输出与带噪图之间计算 L2 loss**。
- 在L2 loss 下降的需求下，模型就学成输出平均的效果，即干净图。因此虽然给的真值时带噪图，但网络学到的确实是干净图
- 推广期他网络：只需要改改loss就能用到已知的网络上。（注意很多loss不适用，参考前面表格）

**应用**
**RAW 域 + N2N = 非常好的组合**，条件（噪声零均值、无非线性变换）几乎完美满足。唯一需要注意的是：**光线越暗，效果越差**（后续研究论文的结论）

# 局限

后续的研究，尤其是降噪任务，大部分还是需要真值数据对的。说明本文方案难以获得非常好的效果。本文方案的主要问题是：
1. 苛刻的应用门槛：**同一个场景、两张噪声相互独立的图片**，实际上很多数据满足不了（老照片、医学影像）。同时连拍也会因轻微的抖动、画面物体运动或光照变化造成**两张图的真值一摸一样**这一点不成立。
2. 很多loss 不适用，也有限制
3. camera 降噪领域用的少（数据更好活去），但是医学领域等**数据难获取**场景应用更广泛。
