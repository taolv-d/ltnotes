# YOND

论文：YOND: Practical Blind Raw Image Denoising Free from Camera-Specific Data Dependency
YOND 是 You Only Need a Denoiser 缩写，也是模仿的 xxx is all you need

本文要解决的问题是：对于图像降噪这个难题，往往需要针对不同的传感器（噪声不同）训练专用网络，没有一个泛化非常好的网络能够应对所有场景。按本文的方法，先对图像做一个变换，把噪声分布都变成高斯分布，可以用一个网络来适应所有的噪声。

不过他的思想其实早就有了，比如[[VST]]变换，pmrid中的变换（[[2020 PMRID Megvii Tsinghua]]）。不过这些变换的目的是降低降噪的难度，这样可以用简单网络降噪，用于手机等低算力平台。不过这些工作仍需要针对特定sensor进行标定。

本文解决的问题就是，不需要再每个sensor单独标定噪声，他直接从图像中估计。有了噪声参数就可以VST变换，将复杂分布的噪声变成简单的高斯分布。

不过本文噪声分布估计、VST 变换中的推导等都是近似的，因此降噪效果跟专用模型还有差距（实测相对[[../nn_isp/2025 CANS|2025 CANS]]这些效果要差一些）。

本文算法主要分为三个步骤：
1. 估计噪声（利用平坦区域）
2. VST 变换，转为高斯噪声去噪问题
3. 网络降噪

# 实现
## 噪声估计

噪声估计其实就是在平坦区域计算标准差就行了。难的是如何找到平坦区域（如何区域噪声跟纹理）
不过这个问题其实已经有一套解决的方法论了，可以参考 demoniac 算法的做法，它们面对的问题是相通的：如何区分纹理跟噪声。

本文的做法也是常规的方案：1. 先猛猛降噪（会损失纹理）2. 降噪前后的图做差，得到含纹理噪声方差 3.  自适应阈值等方法把纹理方差部分拉回来一点。

**具体步骤**
1. 粗估计：使用图像标准差+自适应阈值找到平坦区域，以此参数变换图像并调用模型降噪。
2. 利用方差的加性特性：
	- 真实方差=带噪图方差（噪声+部分纹理）- 过度降噪图方差（部分纹理）。
	这样就可以排除一些纹理的干扰，但仍然噪声偏大。这也是本文估计的问题。

**自适应阈值**
1. 平坦区域的方差应该很小。
2. 选出的平坦区域必须满足两个条件：
	- **数量足够多**：否则统计意义不足。
	- **亮度分布足够广**：否则拟合出的噪声曲线只在局部有效，泛化差

## VST 变换

VST 变换这里有一个长期存在的问题，即 VST 变换虽然将噪声的标准差变为固定值，同时也让噪声的**均值不在是0**了。因此经过VST 变换后图像的亮度实际发生了变化。这个偏差在暗部影响严重（不关心颜色，或者黑白相机对亮度准确性要求不高可以考虑忽略）。

为此，需要计算出带噪信号与干净信号的亮度差异，如下为偏差计算公式：
$$
ϵx​=E[fσ​(z)∣x]−fσ​(x)
$$

| 符号           | 含义                               |
| ------------ | -------------------------------- |
| $fσ(⋅)$      | VST 变换函数，带根号的非线性函数               |
| $x$          | 干净信号值                            |
| $z$          | 带噪信号值                            |
| $fσ​(x)$     | 干净信号经过 VST 后的值                   |
| $E[fσ(z)∣x]$ | 在给定 xx 的条件下，带噪信号经过 VST 后的**期望值** |
这个偏差是在 VST 变换域得到的，因此在逆变换前要加上这个偏差。具体的流程如下：
1. 带噪图VST变换
2. VST 域修正偏差+降噪
3. 逆VST 变换

这里步骤 2 先修正偏差，还是先降噪也是有讲究的：
- 先降噪，早期做法，由于偏差本身无法获取真实值，因此又引入了噪声
- 先修偏差，YOND 方法，降噪网络还能修补 偏差引入的噪声

接下来，回答偏差的公式：$fσ​(x)$ 这一项是真实干净信号的VST 变换。显然我们永远拿不到，两种解决方法：
1. 早期方案：用降噪后的图作为真值，但这样会引入系统偏差
2. 本文：直接用带噪图，带噪图中亮度值相近的像素的期望就能近似真值。因此它通过亮度分层来近似（这里的思想很像NLM，BM3D 这种想办法在单帧图构造多帧的思想）

虽然带噪图明显带有一个波动，但是 YOND 论文中证明了他的结论（也是近似，并非严格证明）：
首先对偏差函数做泰勒展开：
$$
e(y)=e(x+n)=e(x)+e'(x)⋅n+高阶微小项
$$
其中：
$x$ 是干净信号
$y$ 是带噪信号。
$n$ 是噪声，均值为0。
$e'(x)$ 不确定，但在 $x$ 这个点它是**常数**。
计算上式的**数学期望**：
$$
E(e(y))=E[e(x)]+E[e'(x)⋅n]+E[高阶项]
$$
这里忽略影响很小的高阶项。第二项由于 $E[n] = 0$，所以 $E[常数×n]=0$，直接消失。
**结论**：虽然用 `y` 算出来的单次偏差$e(y)$是不准的，但**平均来看**，它的期望值完美等于真实偏差$e(x)$

所以，EM-VST 的策略是：
> **我不管单个像素补偿得准不准，我只要保证全图所有相似亮度的像素，平均补偿量是对的。**
