# 退化原因

本文总结以下表格中6点原因， **真实世界的视频退化往往是复合的**

- Transient particulate degradation 是瞬态颗粒物，通常指雨雪等离散的干扰

- Continuous particulate degradation 是连续散射，通常指雾、霾、水下拍摄等。大气散射模型（公式2）：$观测图 = 真实图 × 传输图 + 大气光 × (1 - 传输图)$  传输图随深度指数衰减。去雾和水下增强的核心就是估计传输图和大气光，然后反推出干净图像。

- Low-Light: 暗、噪、糊、闪(AE帧间变化大)四个debuff。而且这四个问题彼此矛盾——提亮就会放大噪声，降噪可能丢失细节，长时间曝光又会引入模糊。这种“按下葫芦浮起瓢”的特性，让它成为视频增强里最难处理的任务之一。

- Atmospheric Turbulence 大气湍流：带来的退化是让画面**几何失真**——不同区域朝着不同方向扭曲，像透过热浪看东西一样。传统方法用幸运成像思路（抓拍湍流暂时稳定的瞬间），深度学习方法则需要建模这种空间可变的几何畸变。

# 视频增强方法
## 传统视频增强方法

传统算法依赖于特定的手工先验或模型假设，在复杂多变的真实场景中往往会失效。而且需要精细调参，处理动态场景时容易产生新伪影。

**1. 模型驱动方法 (Model-driven methods)**  

这类方法会先为图像退化过程建立一个数学模型（比如噪声是怎么来的、雾是怎么散射光的），然后求解（常用ADMM 等迭代优化算法）这个模型的“逆问题”来恢复清晰图像。

- **例子**：暗通道先验（DCP）去雾、Retinex 理论用于低光增强

**2. 基于规则的方法 (Rule-based methods)**  

按预设规则调整像素值，不关心图像是怎么退化的。对整张图（甚至整个视频）应用相同变换，不看内容。速度快、计算简单，但无法适应复杂多变的场景

- **例子**：全局直方图均衡化、固定伽马校正、线性对比度拉伸、色调映射中的对数/Sigmoid 压缩函数。

**3. 按退化类型的具体应用：**

- **去噪**：空间域滤波（中值/高斯/维纳），缺点是会模糊细节。时间域滤波利用帧间冗余，但对运动物体容易产生拖影。空时联合滤波（如 VBM3D）效果更好，但计算复杂度非常高，且在高噪声下性能下降。
    
- **去雾**：基于大气散射模型，用暗通道先验等方法估参数。但在天空等明亮区域容易失效，直接逐帧用到视频上会产生闪烁。
    
- **去模糊**：目标是反解模糊核和原始图像的卷积。如果模糊核（PSF）已知，可以用**维纳滤波**或Richardson-Lucy 去卷积；如果未知（**盲去卷积**），则需要同时估计二者，对噪声敏感，且难区分运动模糊和真实运动。
    
- **大气湍流**：利用湍流随机变化的特性，**在多帧中找到最清晰的“幸运区域”或“幸运帧”，对齐后融合**。这要求场景本身几乎没有运动，否则对齐会失败。
    
- **低光增强**：直方图均衡系列方法容易过增强或引入噪声。Retinex理论将图像分解为光照和反射分量，通过增强反射分量来提亮图像。

## 有监督学习

需要**大量时空完美对齐的配对数据**。问题是没有标准答案的场景（水下、老电影修复）搞不了。


**1. 卷积神经网络 (CNNs)**

- **如何处理时间信息**：
	- 滑动窗口（处理连续的几帧）
	- 3D 卷积（同时处理空间和时间维度）
	- 显式地用光流估计运动并进行运动补偿。

**2. 循环神经网络 (RNNs)**

- **核心能力**：天生适合处理序列数据，通过隐藏状态在帧间传递信息，适合时序建模。

**3. 基于Transformer的网络**

- **核心能力**：注意力机制善于捕捉长距离的时空依赖关系，可以弥补CNN感受野有限和RNN难以解决长序列中梯度消失的问题。

## 无监督学习

**关键术语澄清**

- **无监督学习**：训练时**完全不需要成对的“退化-干净”视频**。可以利用无标签数据、GAN的对抗训练、或者视频内在结构（如时序一致性）。
    
- **自监督学习**：无监督的一个**子集**。监督信号不来自外部标注，而是从数据本身自动生成。常见做法是设计“预置任务”，比如遮住一部分让模型去补全、或者让同一视频的两个增强版保持一致。
    
- **零样本学习**：**测试时**完全没见过这个任务的训练数据。通常依赖预训练模型的先验或利用输入视频自身统计特性进行自适应。

**五种路线介绍**

| 类别          | 监督信号来自哪      | 核心优势          | 核心挑战           |
| ----------- | ------------ | ------------- | -------------- |
| **域翻译**     | 另一个高质量视频域的分布 | 不需要配对，可迁移风格   | 可能产生伪影，分布覆盖有要求 |
| **自监督信号设计** | 数据自身的结构和冗余   | 完全不需要外部数据     | 盲点方法可能丢失信息     |
| **一致性方法**   | 增强前后的不变性约束   | 保持时序稳定和语义完整   | 复杂运动时难以对齐      |
| **退化感知**    | 再退化后与原始输入的差异 | 巧妙地自监督        | 依赖退化模型的可模拟性    |
| **先验方法**    | 物理模型或预训练模型   | 可解释，能处理极低数据场景 | 领域不同时可能不适用     |

### 域翻译：在“低质域”和“高质域”之间搭桥

**核心思想**：不需要配对数据，只需要两个独立的视频集合——一个低质集合（如低光视频），一个高质集合（如正常光视频）。然后用GAN框架把低质视频的外观“翻译”成高质视频的外观。

**关键机制**：

- 通常采用**CycleGAN**式架构：两个生成器（A→B和B→A）、两个判别器。
    
- **循环一致性损失**是关键约束：一张低光图翻译成正常光再翻译回来，应该和原图一样。这个约束保证了内容不丢失。
    
- **身份损失**防止不必要的色调偏移.

**局限**：需要两个视频域的分布有足够的覆盖度；训练双生成器计算量大；基线CycleGAN框架容易产生伪影。扩散模型开始被引入域翻译，但因计算成本问题目前仍主要停留在图像阶段。

---

### 自监督信号设计：从数据内部自己挖掘监督

**盲点和噪声方法**：在预测某一个像素时，网络不允许看到这个像素自己的值，只能靠周围的像素（和相邻帧）来猜。这迫使网络学习真正的去噪，而不是简单地复制输入。

**优化展开方法**：经典优化（如ADMM）是“迭代求解”，优化展开是把每一轮迭代变成一个网络层，整个优化器被“展开”成一个可训练网络。

**深度即插即用**：经典优化框架里有一个“正则化项”，PnP把这个项替换成一个预训练好的去噪器。这样既保留了物理模型的约束力，又借用了深度学习的强大先验。任务无关但需要迭代求解速度慢。

---

### 一致性方法：“变亮可以，但不能变样”

**增强前后，某些东西不能变**。

**时序一致性**：相邻帧在增强后不能出现突兀的变化（闪烁、抖动），需要通过光流来对齐和约束。

**语义一致性**：增强前后，**高层语义信息不能变**——物体的类别、位置、分割掩码应该保持一致。

---

### 退化感知方法：把图像变暗，再比较

**核心思想**：把增强后的视频**重新退化**回去，然后比较再退化结果和原始低质输入，两者越接近说明增强越可信。

1. 低质输入 → 增强网络 → 增强结果
    
2. 增强结果 → 再退化（重新加噪/重新模糊）→ 再退化结果
    
3. 比较“再退化结果”和“原始输入”

---

### 先验方法：让“见过的世界”帮你判断

**物理先验**：用已建立的物理成像模型（如 Retinex）来约束增强过程。

**学习先验**：用在大规模数据上预训练好的模型提供的隐式表征作为先验（蒸馏？）。

- **Deep Image Prior**：网络结构本身就是一种隐式先验（适合生成自然图像）。
    
- **Deep Generative Prior**：用GAN学习大量自然图像的分布。
    
- **扩散模型先验**：用DDPM等扩散模型约束恢复过程。
    
- **CLIP视觉语言先验**：利用CLIP的跨模态知识判断图像是否“自然”。

# 无监督视频增强中的损失函数

以下这些损失不会单独使用，都是组合使用的

| 损失类型      | 监督信号来源      | 核心作用       | 主要风险        |
| --------- | ----------- | ---------- | ----------- |
| **参考保真度** | 动态生成的伪真值    | 像素级保真      | 伪真值本身有误差    |
| **数据一致性** | 已知退化过程      | 物理可逆性约束    | 退化建模不准时失效   |
| **先验损失**  | 人对自然信号的先验知识 | 正则化、引导恢复   | 先验在实际中不一定成立 |
| **对抗损失**  | 判别器的判断      | 提升感知质量、真实性 | 训练不稳定、虚构细节  |

### 参考保真度损失（L1/L2）

这是有监督学习中最基础的损失（L1/L2），但在无监督设定下，它的"真值"不是外部提供的，而是**动态生成的**。

- **L1 损失（MAE）**：对离群点更鲁棒，不容易被异常像素带偏。
    
- **L2 损失（MSE）**：对大误差惩罚更重，收敛更快，但可能过度平滑细节。

无监督场景下的应用方式：

- **Noise2Noise 思路**：用同一场景下两次独立曝光的噪声图互作真值，即使不知道干净图是啥，L2损失的期望最小值也等于干净信号。[[../camera/denoise/2018 Noise2Noise NVIDIA|2018 Noise2Noise]]

### 数据一致性损失

核心思想：**增强结果在经历已知退化过程后，应该能回到原始输入。**

简单形式为：||A(x_enhanced) - y_observed||，其中A是已知退化算子（如模糊核、采样矩阵）。

**空间一致性**

- **空间一致性损失**：约束相邻像素在增强前后的亮度变化应相似，从而保留边缘和纹理，避免局部畸变。
    
- **自监督残差和一致性损失**：同一帧下采样成两个有差异的噪声版本，教去噪子网络预测并减去噪声，完全不需要干净帧。

**时序一致性**

- **短时一致性**：用光流将前一帧的增强结果扭曲到当前帧，与当前帧增强结果计算差异。关键是要在未遮挡区域计算，否则会被运动边界误导。
    
- **长时一致性**：整个视频序列中物体外观应保持稳定，防止颜色和纹理在长序列中累积漂移。

### 先验损失

将人类对自然信号特性的先验知识编码为数学约束，可以看作一种手工设计的正则化。

| 先验类型                   | 含义                      | 典型应用                           |
| ---------------------- | ----------------------- | ------------------------------ |
| **平滑先验**               | 真实信号（如光照图）在大部分空间都是平滑变化的 | 总变分损失、L1/L2梯度惩罚，常用于Retinex光照估计 |
| **稀疏先验**               | 自然信号在某个变换域（如小波、梯度域）是稀疏的 | L1惩罚梯度，减少振铃效应和噪声放大             |
| **Dark Channel Prior** | 无雾图像的每个局部窗口里总有一个通道趋近于零  | 引导去雾网络不过度提亮，抑制伪影               |
| **色彩恒常先验**             | 增强前后颜色应保持稳定             | L1/L2失真的色差惩罚                   |
| **边缘感知平滑损失**           | 平滑但不抹平边缘                | 用输入图像的梯度来加权平滑损失，在边缘处放松约束       |

### 对抗损失 (GAN)

- **标准形式**：生成器要把低质图变成"看起来像高质量域"的图，判别器要能区分"真高质量图"和"生成出来的高质量图"。为避免生成完全不真实的内容，通常搭配**循环一致性损失**（CycleGAN）和**身份损失**。

# 合成失真数据方法

实际困境
- **无监督方法训练时不需要真值，但评测时还是需要配对数据来算PSNR、SSIM这类客观指标。**
- 有些方法想用合成数据来做“伪有监督”训练。

**合成模糊**

- **运动模糊**：核心做法是对高帧率视频做帧平均来模拟慢快门效果。

- **失焦模糊**：DaBiT是代表性工作，用深度估计模型（DepthAnything）获取深度图，再根据像素离焦平面的距离施加不同强度的空间可变高斯模糊。

- **低光模糊**：LOL-Blur数据集专门做低光+去模糊联合任务，先模拟低光效果，再做模糊合成。

**合成噪声**

- **RAW域噪声**：用物理模型（泊松-高斯噪声模拟散粒噪声和读取噪声），有学者发现用Tukey-Lambda分布替代高斯分布能更准地建模读取噪声。也有方法用GAN和归一化流来学习复杂的sRGB域噪声特性。

- **视频噪声合成**

**合成低光照**

- **传统做法**：直接做线性/伽马变换 $y=\alpha x^\gamma+\beta$，效果不自然。

- **深度学习方法**：EC-Zero-DCE是一个代表，它反着用Zero-DCE的思路，通过可控的反向曲线把正常光视频映射成低光照视频，还能控制暗的程度。


**合成光衰减（雾霾和水下）**

- **雾霾合成**：基于大气散射模型，传输图t(x)随深度指数衰减。

- **水下合成**：模型更复杂，因为不同波长光的衰减率不同。UWNR模型用深度图+光场图来渲染，且让所有帧共享同一个光场，保证时序一致性。

**合成大气湍流**

- **传统做法**：用随机运动图扭曲干净图再高斯模糊，或用Zernike多项式模拟相位畸变。

- **真实热源尝试**：用煤气灶、多个加热室产生真实热浪，但传播距离太短，不够真实，且场景静止缺乏多样性。

- **局限**：这种基于物理的合成方法在弱湍流下还行，到了中强湍流就失真了，容易高估模糊和空间相关性。

# 挑战和开放问题

**1. 时序一致性** ：复杂运动、遮挡、场景突变时易产生鬼影和扭曲。

**2. 多退化联合处理** ： 现实中的视频往往同时受多种退化影响，针对单一退化设计的模型效果不行

**3. UGC视频质量提升** ： 用户随手拍的视频退化方案没有。

**4. 评估困难** ： 没有真值就不好评估。现有的无参考图像质量指标不太行。

**5. 时序质量评估** ：最常用的时序稳定性指标是光流扭曲后算MSE，但只在非遮挡区域计算。

**6. 实时应用**
