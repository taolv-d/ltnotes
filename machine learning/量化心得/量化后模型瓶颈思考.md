# 量化后数据类型对速度的影响

当前无论是CPU，GPU，NPU大体还是遵守冯诺依曼架构，NPU有所改进，但仍然需要从内存加载数据、权重。因此对模型推理速度的主要影响就可以分为两个主要因素：
1. 计算单元速度
2. 内存传输速度
对于计算单元的速度，共识是浮点比定点要慢，不过浮点运算可以通过流水线隐藏计算时间。但是这也会造成功耗、芯片面积（成本增加）。高通的比较了int8 跟fp8，结论就是：如果你追求**最佳的精度与效率权衡**，请使用 **INT4 / INT8 / INT16** 量化，而不是 FP8。[[2303.17951] FP8 versus INT8 for efficient deep learning inference](https://arxiv.org/abs/2303.17951)

其次，对于 int32 int16 int8 int4 的比较需要分器件来看：
	1. 通用CPU，他的计算单元（ALU）吃的就是32位，哪怕你传入的是int8量化的参数，真正运算时也会高位补零到32位。
	2. 对于针对推理优化的硬件，如NPU，他的计算单元能支持1个int32, 2个int16 或者4int8，此时用int8量化的速度。
	3. 对于int4，目前没有看到有专门优化的硬件支持，因此计算速度上其实不会提升

对于传输速度，完全跟量化的bit数有关系了，int4的带宽需求是int8的一半，这也就是LLM喜欢用int4量化的原因，他们的瓶颈往往是大量参数的传输

# 模型架构的影响

模型架构的对速度也有很大的影响，架构决定了模型对内存带宽和计算计算资源两部分的需求。其中cnn这种计算密集型的网络，往往瓶颈出现在计算单元。transformer 架构则对带宽要求更大，很多研究聚焦到这里。一下是关注的一些模型的优化：
