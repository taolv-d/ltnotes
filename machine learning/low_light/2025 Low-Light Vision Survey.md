原文：Deep Learning for Low-Light Vision: A Comprehensive Survey

本文介绍到2025年为止的低光 视觉模型。**更好的视觉增强效果，并不一定能带来更好的识别准确率。 真正有效的方法，是那些将增强和识别联合优化的端到端方案**，让增强直接服务于识别任务，而不是服务于人眼。

**未来方向**

1. **统一模型**：打破目前“一个任务一个模型”的碎片化现状，用一个模型应对多种退化。
2. **预训练大模型**：将LLM/VLM的丰富语义知识蒸馏到低光任务中，或注入退化感知能力。
3. **多模态信息**：融合红外、事件相机、LiDAR等，弥补单模态的物理局限。
4. **高维数据**：从2D图像拓展到3D点云与光照建模。
5. **结合GSP与GNN**：用图信号处理和图神经网络，捕捉复杂局部与全局关系，提升鲁棒性。


**low-light vision 主要分为 增强、检测和其他应用三方面。**

# 增强

按**学习范式**来分成了四个阶段：**从依赖昂贵配对数据的有监督学习，逐渐走向更轻量、更灵活的无监督和零样本学习**。

## 模型驱动

先建立一个关于图像如何变暗的物理/数学模型，再想办法求解模型的参数。

- **传统方法 (3类)**：
    - **直方图均衡**
    - **统计模型方法**：利用局部对比度、相机响应模型等统计先验，但一旦偏离假设分布就失效。
    - **传统Retinex方法**：假设图像=光照×反射，分解后把反射分量当作增强结果，但真实噪声会被当作光照变化而保留下来。

## 数据驱动

|            | 监督学习        | 半监督学习        | 无监督学习   | 零样本学习      |
| ---------- | ----------- | ------------ | ------- | ---------- |
| **数据**     | 配对          | 少量配对 + 大量非配对 | 两组独立非配对 | 仅低光图       |
| **监督信号**   | GT正常光图      | GT + 判别器     | 判别器     | 无参考先验Loss  |
| **核心Loss** | L1/L2重建Loss | 混合Loss       | 对抗Loss  | 自监督/先验Loss |
| **泛化能力**   | 弱（过拟合合成数据）  | 中            | 中       | 强          |
| **性能上限**   | 高（但局限在训练分布） | 中高           | 中       | 中低         |
### 一、 有监督学习：性能最强，但数据最贵

1. **端到端方法**
    - **核心逻辑**：纯数据驱动，直接学习从暗图到亮图的映射，不依赖任何光学先验知识。
2. **基于Retinex理论的方法** 
	- [[Retinex]]
3. **基于频率的方法**
    - **核心逻辑**：新兴视角，利用傅里叶变换等工具，发现亮度信息主要在"振幅"分量，噪声和细节在"相位"分量，然后分而治之。
### 二、 无监督学习：打破数据依赖

- **核心逻辑**：只用非配对的数据进行训练，用 GAN 来恢复。
- **优缺点**：数据好获取，但继承 GAN 的问题

### 三、 半监督学习：折中路线

- **核心逻辑**：少量配对数据引导 +  海量非配对数据泛化

### 四、 零样本学习：最轻量、最灵活、最前沿

- **核心逻辑**：**连非配对数据都不需要**。它只拿低光照图本身，通过精心设计的、不依赖参考图的**无参考损失函数**，来引导网络学习增强。

- **性能与效率对比**：它们通常不使用复杂的编解码器架构，因此**模型极小、推理极快**，很适合手机等端侧设备。

# 检测

目前的分类实际是在不同的角度弥补：**低光信息和正常光模型之间的"语义鸿沟"**

| 方法类别      | 核心思想        | 优势          | 局限            |
| --------- | ----------- | ----------- | ------------- |
| **预处理式**  | 先增强后检测      | 简单直观，易于部署   | 增强和检测目标不一致    |
| **特征融合式** | 特征空间融合和联合优化 | 性能最强，直接服务检测 | 网络设计复杂        |
| **图像暗化式** | 合成低光训练数据    | 绕开标注不足问题    | 合成数据与真实数据有gap |
| **域自适应式** | 迁移白天知识到黑夜   | 无需暗光标注，通用性好 | 极端场景下仍难弥合     |
| **多模态式**  | 融合红外、LiDAR等 | 极暗光下最鲁棒     | 硬件成本高，需多模态对齐  |

### A. 预处理式方法

**核心逻辑**：先增强，再检测

- **做法**：先过增强网络，再送入检测器。

- **增强和检测的目标不一致**，增强左右可能不理想

### B. 特征融合式方法

**核心逻辑**：直接在**特征空间里融合**增强和检测两支

- **做法**：设计一个并行或级联的结构，让增强网络和检测网络共享特征，端到端联合优化。
- **评价**：**目前性能最强的路线之一**。但网络设计相对复杂。

个人理解：也可以理解为复杂的**联合优化**让网络自己学

### C. 图像暗化式方法（生成数据方法）

**核心逻辑**：训练时把**正常数据集变暗**，用合成数据对来训练模型。

**评价**：**合成的暗图与真实暗图的噪声、退化和分布之间依然存在gap**，在真实场景下泛化能力可能不足。

### D. 域自适应式方法（迁移到低光）

**核心逻辑**：把在正常光照上训好的检测器，迁移到低光照的目标域，缩小白天的知识和黑夜的分布差距。

- **做法**：通过对抗学习对齐特征分布，或通过自训练生成伪标签。

- **核心技术**：
    - **域不变特征学习**：在特征提取器中插入域判别器和梯度反转层（GRL），强迫网络学出对光照不敏感的特征。
    
    - **域翻译**：用CycleGAN等把目标域图像风格翻译成源域风格，再用源域的检测器检测。
    
    - **知识蒸馏**：如**YOLO in the Dark**，用多个教师模型共同指导学生在低光下的检测。
    
    - **自训练**：用源域训练好的模型对低光图生成高置信伪标签，再迭代更新。
    
- **评价**：在处理有标注的白天数据和完全无标注的暗光数据时，这是最有效的方法之一。但对极暗光下的极端场景，分布差异过大时仍难完全弥合。

### E. 多模态式方法

**核心逻辑**：既然可见光差得离谱，不如直接引入其他模态来降维打击。

- **做法**：融合可见光、红外、LiDAR等多种传感数据，利用模态间的互补性提升暗光检测鲁棒性。

# 其他低光视觉任务

**语义分割**：在黑暗中分辨道路、行人、车辆等不同类别，比检测更精细，要求像素级的理解。白天训好的模型直接用于暗光，几乎必然会失效。

**目标跟踪**：语义分割，白天模型不适用夜间。不过可以用红外增强

**人体姿态估计**：极暗光下，人体轮廓模糊，信噪比极低，动作识别更加困难。
  
**运动目标检测（背景分离）**：固定摄像头下，光照变化、影子、坏天气都会干扰前景提取。
