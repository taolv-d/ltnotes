# 训练范式

本文提到的三种训练范式如今已有有比较广泛的应用，参考[[review 2025 Low-Light Vision]] [[review 2025 Video Quality Improvement]]

**监督式**：需要 (LR，HR) 图像对，将生成图跟高清图送给判别器识别真假

**无监督**： 没有成对的（LR, HR），利用循环一致性CycleGAN

```
         LR域                      HR域
    ┌─────────┐   生成器G    ┌─────────┐
    │   LR    │ ──────────→ │  G(LR)  │
    └─────────┘             └─────────┘
         ↑                        │
         │        判别器D          ▼
    ┌─────────┐   生成器F    ┌─────────┐
    │  F(HR)  │ ←────────── │    HR   │
    └─────────┘             └─────────┘
    
    循环一致性约束：F(G(LR)) ≈ LR
                  G(F(HR)) ≈ HR
```
这里有三个网络
- 生成器G 挫图生成高清图
- 生成器F 高清图退化到挫图
- 判别器D判断G生成的图与真实图中那个是假的（这里比较的不是同一对图）

**半监督**：半监督可以认为是 监督+无监督的混合，他的数据通常是少量成对数据+大量不配对的数据（即只有挫图，或者只有高清图）。
- 训练可以混着两种范式，对于配对数据走监督式；不配对走无监督部分。
- 或者先用配对数据初始化网络，随后用无配对数据来泛化

# 技术改进侧重点

|改进方向|核心问题|解决思路|
|---|---|---|
|**① 网络架构**|生成器/判别器不够强|设计更深、更聪明的网络结构|
|**② 先验知识**|纯数据驱动信息不足|引入额外的领域知识|
|**③ 损失函数**|像素Loss不够"聪明"|设计更贴合人眼/任务的Loss|
|**④ 多任务学习**|只做SR太单一|同时解决多个相关问题|
