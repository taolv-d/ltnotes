## 第五步：进入 Transformer Encoder

以 **ViT-Base** 为例：L=12 层，每层结构相同。

### 一个 Encoder Block 内部（维度变化）

输入 `x`：`(1025, 768)`

#### 5.1 多头自注意力（MSA）

- **LayerNorm**：维度不变 `(1025, 768)`
- **分成 12 个头**：每个头 `768/12 = 64` 维
- **Q, K, V 矩阵**：每个是 `(1025, 64)`（12 个头并行）
- **注意力计算**：
  - Q·K^T 形状：`(1025, 1025)`
  - 对行 Softmax → 权重矩阵
  - 权重 × V → `(1025, 64)`
- **合并头**：`(1025, 64×12) = (1025, 768)`
- **输出投影**：`(1025, 768)` → `(1025, 768)`
- **残差连接**：`x + 输出` → `(1025, 768)`

> 核心：**1025 个 token 之间两两做注意力**，序列长度不变，维度不变。

#### 5.2 MLP 层

- **LayerNorm**：`(1025, 768)`
- **第一层全连接**：`768 → 3072`（4 倍扩展）  
  形状：`(1025, 3072)`
- **GELU 激活**
- **第二层全连接**：`3072 → 768`  
  形状：`(1025, 768)`
- **残差连接**：`(1025, 768)`

一句话总结：**`[CLS]` 是一个没有物理意义、完全可学习的“信息汇聚点”。它通过自注意力机制，主动吸收所有图像块的信息，最终代表整张图的全局特征。**
对于 `[CLS]` 令牌：

- 它是一个**随机初始化的、可学习的向量**。
    
- 它的 **查询** 是经过训练，学会**对所有图像块一视同仁地关注**。
    
- 它的输出 = 所有图像块信息的 **加权和**，权重由它和每个图像块的关系决定

非常好的追问。位置编码是 ViT 能够理解图像空间结构的核心。因为没有它，Transformer 会把一张打乱 Patch 顺序的图当成原图。

我们彻底展开，从**为什么需要**、**常见方案**、**ViT 的选择**，到**具体计算与直观理解**。

## 一、为什么需要位置编码？

Transformer 的自注意力机制是**排列不变**（Permutation Invariant）的。

- 输入序列 `[A, B, C]` 和 `[C, B, A]` 经过 Self-Attention（不考虑位置时），输出集合相同，只是顺序变了。
- 对于图像：Patch `(猫脸, 猫耳, 背景)` 和 `(背景, 猫耳, 猫脸)` 在无位置信息时会被视为**同一个无序集合**。

**但图像显然是依赖空间结构的**——猫耳应该在猫脸的左上或右上，而不是随机位置。所以必须显式注入位置信息。

---

## 二、主流位置编码方案对比

| 方案               | 原理           | 是否可学习 | 是否绝对 | 代表模型           |
| :--------------- | :----------- | :---- | :--- | :------------- |
| **正弦/余弦编码**      | 固定函数生成，频率不同  | 否     | 是    | 原始 Transformer |
| **可学习 1D 编码**    | 每个位置一个可训练向量  | 是     | 是    | **ViT**, BERT  |
| **可学习 2D 编码**    | 分别学习行、列编码再组合 | 是     | 是    | DeiT, BEiT 变体  |
| **相对位置编码**       | 编码两两之间的偏移    | 是     | 否    | Swin, T5       |
| **RoPE（旋转位置编码）** | 用旋转矩阵编码相对位置  | 否     | 相对   | LLaMA, GPT 系列  |

**ViT 不用正弦**：1D结构无法建模行列关系

**ViT 不用 2D**: 实验发现 1D 和 2D 性能差不多（对于分类任务）, 1D隐式学到了2D信息。

**ViT 不用 相对位置编码**: 计算复杂度更高

**`[CLS]` 的位置编码**

`[CLS]` 是序列的第一个位置（位置 0），它也有自己独立的位置编码。

- 它的位置编码不与任何真实图像块对应。
- 训练后，`[CLS]` 的编码通常会与其他所有 Patch 编码保持适度的距离（不太近也不太远），以便它能够**均匀关注**全局。

## 第一步：Q/K/V 是什么？从哪来？

在自注意力中，**同一组输入**通过**三个不同的线性变换**得到三个新矩阵：  
- **Q (Query)**：我要查什么  
- **K (Key)**：我有什么可被查的标签  
- **V (Value)**：我实际要提取的信息  

输入：`x` 形状 `(1025, 768)` —— 这是经过“位置编码相加”后的结果。

### 三个线性层（不带 bias 简化理解）

```python
W_q = (768, 768)  # 可学习权重
W_k = (768, 768)
W_v = (768, 768)

Q = x @ W_q  # (1025, 768)
K = x @ W_k  # (1025, 768)
V = x @ W_v  # (1025, 768)
```

> 此时 Q/K/V **形状完全相同**，但数值不同（因为 W_q,W_k,W_v 不同）。

---

## 第二步：为什么要“多头”？—— 把 768 维拆成 h 个小组

单头注意力的局限：只能学习**一种**注意力模式。多头注意力把 768 维拆成多个“子空间”，每个子空间独立学习不同的特征关联。

**超参数**：  
- `d_model = 768`  
- `num_heads = 12`  
- `d_k = 768 / 12 = 64`

### 对 Q 做拆分（K, V 同理）

Q 形状 `(1025, 768)`  
→ reshape 成 `(1025, 12, 64)`  
→ 转置为 `(12, 1025, 64)` 方便并行计算

```python
# 伪代码
Q = Q.view(1025, 12, 64).transpose(0, 1)   # (12, 1025, 64)
K = K.view(1025, 12, 64).transpose(0, 1)   # (12, 1025, 64)
V = V.view(1025, 12, 64).transpose(0, 1)   # (12, 1025, 64)
```

现在你有了 12 个独立的注意力头，每个头处理 `(1025, 64)` 的序列。

---

## 第三步：每个头内部的计算（单头示例）

取第 `h` 个头的 Q_h, K_h, V_h，形状都是 `(1025, 64)`。

### 3.1 计算注意力分数

```python
scores = Q_h @ K_h.T   # (1025, 1025)
```

- 第 i 行第 j 列的值 = 第 i 个位置的 Query 与第 j 个位置的 Key 的点积  
- 值越大 → 第 i 个位置越应该“注意”第 j 个位置

### 3.2 缩放（Scale）

```python
scores = scores / sqrt(64) = scores / 8
```

- 为什么要缩放？当维度较大时，点积值会很大，softmax 会进入梯度极小的饱和区。除以 `sqrt(d_k)` 让方差稳定在 1 左右。

### 3.3 掩码（可选）

ViT 中**没有**因果掩码（因为是图像，不是自回归）。所有位置相互可见。

### 3.4 Softmax

```python
attn_weights = softmax(scores, dim=-1)   # (1025, 1025)
```

- 每一行求和 = 1  
- 代表每个位置的注意力分布

### 3.5 加权求和 Value

```python
head_output = attn_weights @ V_h   # (1025, 64)
```

- 第 i 行 = 对所有 j 位置的 V_h[j] 按注意力权重加权求和  
- 这就是“从整个序列中按需提取信息”的结果

---

## 第四步：合并多头输出

每个头输出 `(1025, 64)`。

### 4.1 拼接

```python
all_heads = [head_0, head_1, ..., head_11]  # 每个 (1025, 64)
concat = torch.cat(all_heads, dim=-1)       # (1025, 768)
```

### 4.2 输出投影

```python
W_o = (768, 768)  # 可学习权重
multihead_out = concat @ W_o                 # (1025, 768)
```

- 这一步让不同头的信息混合、重新分配维度  
- 最后残差连接：`x = x + multihead_out`

| 步骤 | 操作 | 形状变化 |
| :--- | :--- | :--- |
| 输入 x | 来自上一层 / 位置编码 | (1025, 768) |
| 线性变换 | Q = x @ W_q | (1025, 768) |
| | K = x @ W_k | (1025, 768) |
| | V = x @ W_v | (1025, 768) |

| 合并头 | concat 12 个头 | (1025, 768) |
| 输出投影 | out = concat @ W_o | (1025, 768) |
| 残差连接 | x = x + out | (1025, 768) |

## 容易卡住的细节（帮你提前排雷）

### Q1：Q/K/V 究竟是“从哪来”的？

**不是从外部来，而是输入 x 经过三个不同线性层变换而来。**  
同一个 x 生成三份不同的“身份”：  
- Q 主动问  
- K 被动被问  
- V 被提取的内容

### Q2：为什么 Q 和 K 要分开？

如果 Q=K，注意力矩阵必然对称，表达能力下降。  
Q 和 K 不同 → 可以学习**非对称**关系：位置 i 关注 j，不要求 j 也同等关注 i。

### Q3：多头到底是并行还是串行？

**完全并行**。矩阵运算一次性算出所有头的 Q/K/V，只是通过 reshape 和 transpose 分组。

### Q4：每个头的 W_q 是独立的吗？

是的。完整写法：

```
W_q = (768, 768) 拆成 12 个 (768, 64) 逻辑上分别是 W_q0..W_q11
```

但在实现中常把 12 个头合并成一个大的 (768, 768)，再通过 view/split 分开。

## 代码级理解（PyTorch 风格）

```python
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=768, num_heads=12):
        super().__init__()
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, x):
        # x: (batch, seq_len, d_model)
        batch, seq_len, _ = x.shape

        # 1. 线性变换
        Q = self.W_q(x)  # (B, S, D)
        K = self.W_k(x)
        V = self.W_v(x)

        # 2. 拆分多头
        Q = Q.view(batch, seq_len, self.num_heads, self.d_k).transpose(1, 2)
        K = K.view(batch, seq_len, self.num_heads, self.d_k).transpose(1, 2)
        V = V.view(batch, seq_len, self.num_heads, self.d_k).transpose(1, 2)
        # 现在形状: (B, H, S, D_k)

        # 3. 注意力
        scores = torch.matmul(Q, K.transpose(-2, -1)) / (self.d_k ** 0.5)
        attn = torch.softmax(scores, dim=-1)
        out = torch.matmul(attn, V)  # (B, H, S, D_k)

        # 4. 合并头
        out = out.transpose(1, 2).contiguous().view(batch, seq_len, self.d_model)
        # (B, S, D)

        # 5. 输出投影
        out = self.W_o(out)
        return out
```

## 直觉理解：多头注意力在做什么？

把 **768 维空间**分成 **12 个 64 维的“视角”**：

- 头 1 可能学习**颜色注意力**：红色区域关注红色区域  
- 头 2 可能学习**边缘注意力**：垂直边缘关注垂直边缘  
- 头 3 可能学习**类别注意力**：猫头关注猫身体  
- 头 4 可能学习**空间位置注意力**：近处互相关注  

最终拼接并用 W_o 混合这些视角 → 综合理解。
