
# V1
[[ViT 1]] 是一次性让所有图像块"两两对视"；而 Mamba 是让图像块排成一队，依次走过一个"有记忆的状态空间"，边走边更新记忆，最后用这个记忆代表整幅图。

## 总体流程

对 **512×512 彩色图**，Mamba 处理的总体流程如下：

| 步骤          | ViT 的做法                               | Mamba 的做法                      |
| :---------- | :------------------------------------ | :----------------------------- |
| 1. 图像分块     | 切成 16×16 的 Patch，共 1024 个             | **完全相同**（也可用其他大小）              |
| 2. 线性投影     | 每个 Patch 映射到 D 维向量                    | 完全相同                           |
| 3. 位置编码     | 加一个可学习的位置向量                           | 通常加上（ViM 等变体使用）                |
| **4. 序列处理** | 所有 Patch **同时输入** Transformer，两两计算注意力 | Patch **依次输入** SSM，边走边更新隐藏状态   |
| 5. 全局表征     | 通过 `[CLS]` token 聚合                   | 通过**最终隐藏状态**或 `[CLS]` token 聚合 |
| 6. 复杂度      | **O(N²)**，N 是 Patch 数量（特征块两两交互）       | **O(N)**，线性增长（特征块按顺序依次交互）      |
| 7. 规模放大     | 不容易饱和                                 | 容易饱和                           |

纯 Mamba 模型（无蒸馏、无混合）在**模型规模扩大时性能饱和**较快——从 25M 参数增加到 75M 参数时，精度提升很小，而 ViT 在更大规模下仍能持续提升 。

---

**太长不想看**
1. ViT N个patch两两交互复杂度太高，参考RNN做法，依次过一遍所有patch，选择重要的信息记录下来。 即 $h_{t+1} = Ah_t + Bx_t$  对应输出 $y=Ch_t$
2. 单次遍历，**第 t 个位置的输出 y_t，只能看到位置 ≤ t 的输入信息**。因此搞了双向，甚至四向

## 状态空间模型 (State Space Model, SSM)

SSM 的数学形式其实很简洁。假设我们有一个隐藏状态 `h(t)`（可以理解为"记忆"），输入是 `x(t)`（当前看到的图像块），输出是 `y(t)`：

```math
h'(t) = A·h(t) + B·x(t)   (隐藏状态的更新)
y(t) = C·h(t)              (从隐藏状态产生输出)
```

- `A`（状态转移矩阵）：决定"记忆如何随时间自然衰减/演化"——比如之前的记忆是保留 90% 还是 50%
- `B`（输入投影）：决定"当前输入的图像块对记忆的影响有多大"
- `C`（输出投影）：决定"从当前记忆中提取哪些信息作为输出"

在处理离散的图像块序列时（第 1 个块 → 第 2 个块 → ... → 第 1024 个块），SSM 的工作方式是。最终的 `h1024` 包含了**前 1024 个图像块的全部信息**（被 A 矩阵"有选择地记住和遗忘"过）。

## Mamba 的创新（S6 选择性扫描机制）

传统的 SSM 有个大问题：**A、B、C 三个矩阵对所有输入都是一样的**。

这意味着：无论当前图像块是"猫耳朵"还是"天空背景"，它对记忆的更新方式完全相同。这不合理——我们显然希望模型能**根据内容决定"记住多少、遗忘多少"**。

Mamba 的核心创新是 **S6 选择性扫描机制 (Selective Scan Mechanism)**：让 B、C 矩阵变成**输入的函数**：

```math
B = Linear_B(x)
C = Linear_C(x)
Δ = Linear_Δ(x)
```

也就是说：
- 如果当前块是"重要的"（比如猫脸），B 会变大 → 更多信息写入记忆
- 如果当前块是"冗余的"（比如纯色背景），B 会变小 → 更少信息写入记忆

这个"选择性"就是 Mamba 名字的由来：它能根据输入内容，**动态决定信息流**。


**初始状态**：

输入 SSM 的序列长度为 1025，每个 token 是 768 维。但 SSM 内部处理时，涉及一个**隐藏状态**的维度，记作 `N`（通常是 16 或 64，比 D=768 小得多）。
```python
h = [0, 0, ..., 0]  # 形状 (N,)，初始为空
```

**处理第 1 个 token**（`[CLS]` 或第一个 Patch）：
```python
x1 形状: (768,)

# Step 1: 根据输入计算 Δ, B, C（选择性机制的核心）
Δ = Linear_Δ(x1)   # 标量或小向量，决定"步长" 实际Δ = τ_Δ( Linear_Δ(x₁) ) 
	- `Linear_Δ`、`Linear_B`、`Linear_C` 都是可学习的线性层（权重矩阵 + 可选 bias）
	- `τ_Δ` 通常是 **softplus** 函数：`τ_Δ(a) = log(1 + exp(a))`，确保 Δ > 0
B = Linear_B(x1)    # 形状 (N,)
C = Linear_C(x1)    # 形状 (N,)

# Step 2: 离散化——将连续的 A、B 转换为离散的 A̅、B̅
A̅ = exp(Δ · A)     # A 是固定的 (N, N) 矩阵，mamba中通常也是可学习的
B̅ = (ΔA)^{-1}(exp(ΔA) - I) · ΔB # I 单位阵 [N,N]
# 第二个常常这样近似：B̅ = (A̅ - I) · (A^{-1}) · B     # 前提是 A 可逆，且 ΔA/exp 可交换

# Step 3: 更新隐藏状态
h_new = A̅ · h_old + B̅ · x1  # 向量形状 (N,) 这里还有一个维度对齐的操作，省略了

# Step 4: 产生输出
y1 = C · h_new              # 标量（一个通道）

# 但实际上 Mamba 是并行处理 D=768 个通道
# 最终输出 y1 的形状是 (768,)
```


**处理第 2 个 token**（使用更新后的 `h`）：
```python
# 用 x2 重新计算 Δ, B, C（选择性！）
Δ = Linear_Δ(x2)
B = Linear_B(x2)
C = Linear_C(x2)

# 离散化，更新隐藏状态
h_new = A̅ · h_old + B̅ · x2

# 产生输出
y2 = C · h_new
```

**处理第 3 到第 1025 个 token**：重复以上过程。

### 双向扫描、多方向扫描

文本是天然单向的（从左读到右），但图像不是。图像没有固定的"起点"和"终点"。

**Vision Mamba (ViM) 的解决方案**：**双向扫描 (Bidirectional Scanning)** 。

前向扫描：从左到右，[CLS] → Patch 1 → Patch 2 → ... → Patch 1024
后向扫描：从右到左，Patch 1024 → ... → Patch 1 → [CLS]
融合两个方向（通常是加权平均或拼接后投影）


**为什么要双向？**

因为图像是 2D 结构。如果只从左到右扫描：
- Patch 1 能看到它右边的所有 Patch
- 但 Patch 1024 看不到任何后续信息（它是最后一个）

这会导致**位置偏见**。双向扫描让每个位置都能"看到"整个序列——前向提供"后面的信息"，后向提供"前面的信息"。

**多方向扫描**

```python
方向1: 左上 → 右下
方向2: 右上 → 左下
方向3: 左下 → 右上
方向4: 右下 → 左上

# 四个方向的输出融合
final = combine(f1, f2, f3, f4)
```
