
# VGG

VGG网络的设计遵循一个非常简单的原则：**整个网络都使用相同规格的“积木”**。

-   **卷积“砖块”**：全部使用 **3x3** 大小的卷积核，步长(stride)为1，填充(padding)为1。这样的设置使得卷积操作**不改变输入图片的尺寸**（高和宽）。
-   **池化“隔板”**：在每组卷积层之间，使用 **2x2** 的最大池化(Max Pooling)层，步长为2。它的作用是**将图片的尺寸缩小为原来的一半**（高和宽各减半）。
-   **激活函数**：所有隐藏层后都统一使用 **ReLU** 激活函数，引入非线性。

### VGG16 的整体架构：五段式结构（13个卷积层 + 3个全连接层）

每个块依次 宽高减半（增大感受野），通道翻倍（提取特征更多），相比大卷积核，小卷积核+更深网络层数 不仅计算效率更高，而且引入更多非线性，增强网络表达能力

最后基层全连接参数巨大，计算效率低

| 网络分段 | 层级组成 | 输入尺寸 (高 x 宽 x 通道) | 输出尺寸 | 核心操作 |
| :--- | :--- | :--- | :--- | :--- |
| **输入层** | - | **224 x 224 x 3** | 同左 | 输入一张RGB彩色图片 |
| **卷积块1** | 2个卷积层 + 1个池化层 | 224 x 224 x 3 | **112 x 112 x 64** | 卷积核数量从3 -> **64**，池化后尺寸减半 |
| **卷积块2** | 2个卷积层 + 1个池化层 | 112 x 112 x 64 | **56 x 56 x 128** | 卷积核数量翻倍到 **128**，池化后尺寸减半 |
| **卷积块3** | 3个卷积层 + 1个池化层 | 56 x 56 x 128 | **28 x 28 x 256** | 卷积核数量翻倍到 **256**，池化后尺寸减半 |
| **卷积块4** | 3个卷积层 + 1个池化层 | 28 x 28 x 256 | **14 x 14 x 512** | 卷积核数量翻倍到 **512**，池化后尺寸减半 |
| **卷积块5** | 3个卷积层 + 1个池化层 | 14 x 14 x 512 | **7 x 7 x 512** | 卷积核数量保持 **512**，池化后尺寸减半 |
| **分类器** | 3个全连接层 | 7 x 7 x 512 (展平为向量) | **1000个类别分数** | 通过4096->4096->1000的神经元连接进行分类 |

注意：传统VGG的卷积层只有 conv+ReLU，没有BN，不过后续的改进有加入BN

# AlexNet

AlexNet 是深度学习历史上的一个里程碑，它在 2012 年的 ImageNet 图像识别竞赛中一举夺冠，大幅降低了错误率，从而开启了深度学习在计算机视觉领域的热潮。作为 **Backbone（骨干网络）**，它主要负责从输入图像中提取特征，其经典结构包含 **5个卷积层**和 **3个全连接层**，共8层网络。

| 层（Layer） | 类型（Type） | 输入尺寸（Input Size） | 卷积核/池化核（Kernel） | 步长（Stride） | 输出尺寸（Output Size） | 主要参数/说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Input** | 输入层 | - | - | - | 227×227×3 | 实际输入尺寸通常为 227x227 |
| **C1** | 卷积层 | 227×227×3 | 11×11, 96个 | 4 | 55×55×96 | 后接 ReLU、LRN（局部响应归一化）和最大池化 |
| **Pool1** | 池化层 | 55×55×96 | 3×3 | 2 | 27×27×96 | 采用重叠池化（Overlapping Pooling） |
| **C2** | 卷积层 | 27×27×96 | 5×5, 256个 | 1 | 27×27×256 | 后接 ReLU、LRN 和最大池化 |
| **Pool2** | 池化层 | 27×27×256 | 3×3 | 2 | 13×13×256 | 同上，采用重叠池化 |
| **C3** | 卷积层 | 13×13×256 | 3×3, 384个 | 1 | 13×13×384 | 后接 ReLU |
| **C4** | 卷积层 | 13×13×384 | 3×3, 384个 | 1 | 13×13×384 | 后接 ReLU |
| **C5** | 卷积层 | 13×13×384 | 3×3, 256个 | 1 | 13×13×256 | 后接 ReLU 和最大池化 |
| **Pool5** | 池化层 | 13×13×256 | 3×3 | 2 | 6×6×256 | - |
| **FC6** | 全连接层 | 6×6×256 (9216) | - | - | 4096 | 后接 ReLU 和 Dropout，参数量巨大 |
| **FC7** | 全连接层 | 4096 | - | - | 4096 | 后接 ReLU 和 Dropout |
| **Output** | 输出层 | 4096 | - | - | 1000 | 用于 ImageNet 的 1000 分类，后接 Softmax |

### 为什么会用 AlexNet 做模型的感知 Loss？

*   **AlexNet 的“意外”天赋**：研究发现，像 AlexNet 和 VGG 这样的早期图像分类模型，在评估图像的感知相似度时，表现超越了传统的像素级指标（如 MSE、PSNR），其结果与人类的判断更为接近。这意味着，尽管 AlexNet 没有被专门训练来判断相似度，但它学习到的特征空间本身就具备了这种“感知”能力。
*   **与更先进模型的对比**：有趣的是，一个来自 Google Research 的研究发现，在感知相似度任务上，更现代化、分类精度更高的模型（如 ResNet、EfficientNet 等）反而表现得不如 AlexNet 或 VGG。论文指出，在达到一定的分类精度后，模型分类越准，其提取的特征用于衡量感知相似度的能力反而会下降。
*   **浅层网络的优势**：这可能是因为像 AlexNet 这样相对“较浅”的网络，所提取的特征更侧重于纹理、颜色和形状等视觉外观信息，这与人类的感知方式更契合。而更深、更强的网络，其高层特征可能过度抽象，专注于语义类别区分，反而丢失了细节的感知相似性。研究甚至发现，只训练了几个周期的浅层 ResNet，其感知得分就能媲美 AlexNet。

因此，当我们在图像风格迁移或图像生成任务中使用 **Perceptual Loss** 时，会选择 AlexNet 或 VGG 作为 Backbone，将生成的图像和目标图像分别输入网络，然后计算它们在中间某层（通常是较前期的卷积层）输出特征图之间的差异。这样可以**引导生成的图像在内容和风格上更接近目标图像**，而不是仅仅追求像素间差异的最小化。
