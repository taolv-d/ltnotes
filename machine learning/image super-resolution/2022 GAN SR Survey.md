# 训练范式

本文提到的三种训练范式如今已有有比较广泛的应用，参考
- [[low_light/2025 Low-Light Vision Survey]] 
- [[2025 Video Quality Improvement Survey]]

**监督式**：
- 需要 (LR，HR) 图像对，将生成图跟高清图送给判别器识别真假

**无监督**： 
- 没有成对的（LR, HR），利用循环一致性CycleGAN

```
         LR域                      HR域
    ┌─────────┐   生成器G    ┌─────────┐
    │   LR    │ ──────────→ │  G(LR)  │
    └─────────┘             └─────────┘
         ↑                        │
         │        判别器D          ▼
    ┌─────────┐   生成器F    ┌─────────┐
    │  F(HR)  │ ←────────── │    HR   │
    └─────────┘             └─────────┘
    
    循环一致性约束：F(G(LR)) ≈ LR
                  G(F(HR)) ≈ HR
```
- 这里有三个网络
	- 生成器G 挫图生成高清图
	- 生成器F 高清图退化到挫图
	- 判别器D判断G生成的图与真实图中那个是假的（这里比较的不是同一对图）
- 这里还有一点：**G会偷懒**例如去雨任务中，不管什么输入，生成器G都生成一个晴天的图，这样就骗过了判别器。需要加一个约束，将G生成的结果退化回去跟他的输入比较，如果差异比较大，说明G偷懒了

**半监督**
- 半监督可以认为是 监督+无监督的混合，他的数据通常是少量成对数据+大量不配对的数据（即只有挫图，或者只有高清图）。
- 训练可以混着两种范式，对于配对数据走监督式；不配对走无监督部分。
- 或者先用配对数据初始化网络，随后用无配对数据来泛化

# 技术改进侧重点

| 改进方向      | 核心问题         | 解决思路            |
| --------- | ------------ | --------------- |
| **网络架构**  | 生成器/判别器不够强   | 设计更深、更聪明的网络结构   |
| **先验知识**  | 纯数据驱动信息不足    | 引入额外的领域知识       |
| **损失函数**  | 像素Loss不够"聪明" | 设计更贴合人眼/任务的Loss |
| **多任务学习** | 只做SR太单一      | 同时解决多个相关问题      |

## 网络架构

**生成器**

1. 早期GAN只有几层卷机
2. 提升网络深度 + 残差连接
3. 密集连接（Dense Connection），缓解难训的问题。**代表**：RRDB（ESRGAN的核心）[[image restoration/2021 Real-ESRGAN Tencent|2021 Real-ESRGAN Tencent]]
```text
每一层都接收之前所有层的输出

第1层 ← 输入
第2层 ← 输入 + 第1层
第3层 ← 输入 + 第1层 + 第2层
```

4. 一步到位太难，渐进式生成（每个阶段放大2倍）
```text
×1 ──→ ×2 ──→ ×4 ──→ ×8
  ↓       ↓       ↓
 G1      G2      G3
```

5. 转置卷积（容易产生棋盘格伪影）-> 亚像素卷积
	- 转置卷积：先 w h 扩大2倍，多出来的行列补零，再走卷积。问题是卷积核在不同位置时参与计算的值不同，例如上采样后 [1 0 2 0 3]，这个序列中使用3x3卷积，第一个位置只有一个1参与，第二个位置有1 2 参与，这种不平衡引起啦 artifartifact
	- 亚像素卷积就是我们现在常见的：先卷积增加通道数量，然后pixel shuffle 上采样。

 
**判别器**
1. 早期只输出 一个值：真/假（忽略了细节，只看到全局）
2. 改为输出N* N矩阵，照顾到局部纹理
3. 相对判别器
```
传统： D(real) → 1, D(fake) → 0

相对论： 
   D(real) 和 D(fake) 比
   判断“real 比 fake 真多少”
```

4. 多判别器
```text
生成器 ──┬── 判别器1（看全局）
        ├── 判别器2（看纹理）
        └── 判别器3（看边缘）

必须同时骗过所有判别器
→ 生成结果更全面
```

## 先验知识

GAN 最大的问题：**只见过训练数据，没见过“物理规律”** 因此把一些先验（噪声模型、人脸特征）给到网络可以缓解很多问题，例如避免生成三个眼睛的人。

使用方式分为三种：
1. **作为额外输入**（喂给网络）
2. **作为约束条件**（Loss 拉着他）
3. **作为初始化/引导**（网络起点不一样，先用强先验训练，随后用大量数据泛化）

## 损失函数

像素级Loss（L1/L2）→ PSNR高但图像平滑
感知Loss → 更真实但可能偏离原图

| 损失类型       | 公式思想                  | 作用     | 代表方法      | 原理简介                                    |
| ---------- | --------------------- | ------ | --------- | --------------------------------------- |
| **像素Loss** | \|HR - SR\|           | 保真度    | SRGAN基础   |                                         |
| **感知Loss** | \|VGG(HR) - VGG(SR)\| | 特征相似   | SRGAN改进   |                                         |
| **对抗Loss** | -log(D(SR))           | 真实感    | 所有GAN     |                                         |
| **排序Loss** | 让好的SR"排名"更高           | 优化感知质量 | RankSRGAN | 训练一个排序网络，让人偏好的图得分更高（类似VGG 这种接个小网络做loss） |
| **频域Loss** | \|FFT(HR) - FFT(SR)\| | 恢复高频   | FSLSR     | 比较FFT 后的高频分量，保留的多说明纹理边缘锐利               |
| **质量Loss** | 用IQA模型打分              | 人眼对齐   | GMGAN     |                                         |
| **循环Loss** | \|x - F(G(x))\|       | 内容保留   | CycleGAN系 | 见训练范式-无监督部分                             |

## 多任务学习

真实世界的图像往往有多个问题：
- 低分辨率 + 噪声
- 低分辨率 + 暗光
- 低分辨率 + 模糊

与其"分步处理"，不如**一个网络同时解决多个问题**。早期GAN用多个网络分别解决每个问题。但是现在的已经理所当然用一个网络处理所有问题了。
