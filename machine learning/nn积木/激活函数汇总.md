


# ReLU
`ReLU(x) = max(0, x)`

- 大于 0，原样通过
- 小于 0，直接变 0

图像上看就是左边全砍掉，右边保留直线。

它的优点：

- 简单
- 快
- 很经典

它的问题：

- 负数部分全部变 0，信息直接没了
- 有些神经元可能长期输出 0，出现“死 ReLU”


# ReLU6
ReLU6 是 ReLU的截断版本，目的是为了int8量化场景尽可能少的精度损失
$$ReLU6(x)=min⁡(max⁡(0,x),6)$$
# SiLU (Swish)

`SiLU(x) = x * sigmoid(x)`

它不是一刀切，而是更平滑：

- 很大的正数，差不多原样通过
- 很小的负数，不是直接砍成 0，而是保留一点点
- 过渡很平滑




# h-swish
 MobileNetV3 等轻量级网络中常用的激活函数。

它是 **Swish** 激活函数的一个 **近似、快速、量化友好** 的版本。

h-swish 基于 **ReLU6** 构建，定义如下：

$$h\_swish[x]=x⋅\frac{ReLU6(x+3)}{6}$$

​展开为分段函数：
$$
h\_swish(x)=
\begin{cases}
0, & \text{if}x≤−3 \\
x⋅\frac{x+3}{6}, & \text{-3<x<3} \\
x, & \text{x≥3}
\end{cases}
$$
与 Swish 的对比

| 特性    | h-swish         | Swish |
| ----- | --------------- | ----- |
| 计算复杂度 | 低（加/乘/clamp）    | 高（指数） |
| 平滑性   | 分段平滑（在 ±3 处有拐点） | 处处光滑  |
| 量化友好  | 是               | 否     |
| 典型应用  | 移动端 CNN         | 通用网络  |

# h-sigmoid
h-sigmoid 是 h-swish 的“门控部分”，也是 Sigmoid 的硬近似

$$ h\_sigmoid(x)=\frac{ReLU6(x+3)​}{6}$$
