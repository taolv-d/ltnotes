ViT 流程如下，以ViT-Base/16（Patch=16×16，隐层D=768）为例。

| 阶段              | 形状                  | 备注                                                               |
| --------------- | ------------------- | ---------------------------------------------------------------- |
| 原始图像            | (512, 512, 3)       |                                                                  |
| 切 Patch         | (32, 32, 16, 16, 3) | 32×32 个块                                                         |
| 展平每个 Patch      | (1024, 768)         | 每个 Patch 16×16×3=768                                             |
| Patch Embedding | (1024, 768)         | 线性投影（维度不变）可学习的 Embedding 矩阵(768, 768）                            |
| 添加 `[CLS]`      | (1025, 768)         | CLS 类别tocken，是一个**随机初始化**的**可学习**的向量。它对应的输出就被视为整张图像的全局特征，用来做最终分类 |
| 加位置编码           | (1025, 768)         | 可学习，逐元素相加                                                        |
| Transformer ×12 | (1025, 768)         | 每层形状不变 [[Transformer]]                                           |
| 取 `[CLS]`       | (768,)              |                                                                  |
| 分类头 MLP+Softmax | (1000,)             | 假设 1000 类                                                        |

## 位置编码

Transformer 的自注意力机制是**排列不变**（Permutation Invariant）的：输入序列 `[A, B, C]` 和 `[C, B, A]` 经过 Self-Attention（不考虑位置时），输出集合相同，只是顺序变了。

**但图像显然是依赖空间结构的**，所以必须显式注入位置信息。

**主流位置编码方案对比**

| 方案               | 原理           | 是否可学习 | 是否绝对 | 代表模型           |
| :--------------- | :----------- | :---- | :--- | :------------- |
| **正弦/余弦编码**      | 固定函数生成，频率不同  | 否     | 是    | 原始 Transformer |
| **可学习 1D 编码**    | 每个位置一个可训练向量  | 是     | 是    | **ViT**, BERT  |
| **可学习 2D 编码**    | 分别学习行、列编码再组合 | 是     | 是    | DeiT, BEiT 变体  |
| **相对位置编码**       | 编码两两之间的偏移    | 是     | 否    | Swin, T5       |
| **RoPE（旋转位置编码）** | 用旋转矩阵编码相对位置  | 否     | 相对   | LLaMA, GPT 系列  |

**ViT 不用正弦**：1D结构无法建模行列关系
**ViT 不用 2D**: 实验发现 1D 和 2D 性能差不多（对于分类任务）, 1D隐式学到了2D信息。
**ViT 不用 相对位置编码**: 计算复杂度更高

**`[CLS]` 的位置编码**
`[CLS]` 的位置编码不与任何真实图像块对应。训练后，`[CLS]` 的编码通常会与其他所有 Patch 编码保持适度的距离（不太近也不太远），以便它能够**均匀关注**全局。
